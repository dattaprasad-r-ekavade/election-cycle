# AGENTS.md

## Cursor Cloud specific instructions

This repository is a single **Godot 4.5** game ("Election Cycle"), written in GDScript. There is no package manager, backend, database, or network service — it is a fully offline, client-only single-player game. The only dependency is the **Godot 4.5 engine binary** (installed at `/usr/local/bin/godot`, GL Compatibility renderer). All autoloads/systems live in `systems/`, scenes in `scenes/`, gameplay scripts in `scripts/`, and content data as JSON in `content/`. Main scene is `scenes/main_menu.tscn` (see `project.godot`).

### Engine / setup notes
- If `godot` is missing (e.g. fresh VM without the snapshot), reinstall the standard Linux build: download `Godot_v4.5-stable_linux.x86_64.zip` from the GitHub releases, unzip, and place the binary at `/usr/local/bin/godot` (chmod +x). Match the version in `project.godot` (`config/features` → `4.5`).
- The `.godot/` import cache is gitignored and regenerated by the engine. The startup update script runs `godot --headless --import` to refresh it after pulling changes. Tracked `*.uid` / `*.import` files may be regenerated by an import; do not commit those incidental changes.

### Running the game
- Editor/windowed run (needs a display; `DISPLAY=:1` is available): `DISPLAY=:1 godot --path /workspace`
- Audio: ALSA errors at startup are expected and harmless — the engine falls back to a dummy audio driver.
- Gameplay flow: Main Menu → Campaign select → (intro scene, press Esc to skip) → Character Creation (`> BEGIN CAMPAIGN`) → Town map (arrow keys/WASD to move). Press F3 in-game to toggle the debug overlay.

### Tests (headless, exit code 0 = pass)
- Scene-flow validation: `godot --headless --path /workspace --script res://tests/windows_flow_validation.gd`
- Seed determinism (`tests/seed_test.gd` is an `extends Node` script that runs via `_ready`/`run_test()`): wire it into a scene or a small `SceneTree` runner that adds it as a child and calls `run_test()`, then run headless. There is no committed scene/runner for it.

### Lint / build
- No standalone linter is configured; GDScript parse/type errors surface when the project is imported or run (`godot --headless --import` will report script errors).
- Export builds (optional, Web/Windows/Android per `export_presets.cfg`) require Godot export templates for 4.5 and are not needed to run or test gameplay.
